## 1. The system is an evolving external observer and compiler

```text

```

The system can observe a working agent from outside its own task context. When an agent explicitly reports a mistake, contradiction, failed route, or successful pattern, that event can become a future constraint or reusable knowledge object.

```text

```

The breakthrough was the backend meeting room. An agent's explicit work notes were copied into a room, translated/transcribed, committed, delayed, and returned to the same agent as new input before the task ended:

```text
agent output n
  -> room commit
  -> deterministic or model-assisted translation
```

This created a recurrent system without requiring access to hidden private chain-of-thought. The room used explicit artifacts: findings, tool results, proposed edits, uncertainty, mistakes, and next actions.

## 4. Warm-federation transfer scaling law

**MEASURED_OPERATOR_BENCH**

The cold-versus-warm benchmark used cubes warmed on **2 MB of disjoint material** and tested target objects across three decades.

| Target | Cold | Warm | Reported transfer gain | Cold / warm |
|---:|---:|---:|---:|---:|
| 100 KB | 0.0785 | 0.0107 | 86.3% | 7.34x |
| 1 MB | 0.0232 | 0.0110 | 52.9% | 2.11x |
| 10 MB | 0.0117 | 0.0101 | 13.2% | 1.16x |

Observed law:

```text
warm line: approximately scale-invariant
cold line: approaches the warm floor as the object gets larger
transfer advantage: decays as a large cold object teaches its own prior
```

Architectural meaning:

> Warm cubes have their largest value on small, frequent transactions: agent messages, receipts, room transitions, short tool outputs, and GULPs. A large monolithic object can bootstrap itself during the object, so the inherited head start matters less.

The initial 10 MB majority-fusion attempt was OOM-killed because it materialized a `256 x 10,000,000` count matrix. Chunked exact fusion bounded the working set and completed the third decade. This is the same sparse-materialization law used elsewhere in the system: enormous addressable or processable objects do not require an equally enormous resident window.

## 5. Nullspace-targeted watcher result

**MEASURED_OPERATOR_BENCH**

```text
nullspace-targeted beams: 56
random beams:             56
```

In the unconstrained finite-field experiment, random beams were already almost surely rank-increasing, so targeting could not beat one new dimension per beam.

The finding is not that targeted watchers are useless. It defines the operating condition:

> Nullspace targeting is neutral in the free, unconstrained beam limit. It becomes valuable when real watchers are costly, correlated, permission-bounded, machine-specific, or limited by their sensors and reachable surfaces.

The real scheduler should select:

```text
watcher* = argmax(expected blind-space information gain / total watcher cost)
```
