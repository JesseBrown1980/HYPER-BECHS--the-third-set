## Agent-type composition

The same 8-byte root atom can address or govern different agent classes:

1. **Real paid/subscription agents** - paid model/tool seats, still representable as Host8 handles/stubbed-room routes.
2. **Real free subagents and sub-subagents** - OpenCode/Hermes-style free/provider lanes and spawned subagent work units.
3. **Logical agents** - virtual pointer, packet, room, route, and decision-pressure agents that may not correspond to a live paid/free call.

Do not collapse these. A 100B run can contain logical route generation and still harvest real free-agent outputs.

## Nested reflection shape

The simplest live reasoning unit is a three-agent reflective cell:

1. **Agent 1** works.
2. **Agent 2** reviews and predicts Agent 1's likely next move from its output. This is self-reflection / predictive coding, not just a critic.
3. **Agent 3** reads both outputs and queries the fabric through Hilbra/recall/remote-control lanes.

The rung above them is a supervisor, and any rung can be watched by a higher rung. The system is infinitely nestable because correction/observation can recurse, while consent does not recurse: scale/fire stays anchored at the human/operator apex.

## Why watcher-gated

Recurrence alone is not enough. The root law is:

> recurrence + correction against real input = cognition

The watcher recomputes or independently checks the agent's output. A child is accepted only when its reported state agrees with the watcher/fabric truth under the relevant gate. This is the N-Nest-Prime correction primitive.
