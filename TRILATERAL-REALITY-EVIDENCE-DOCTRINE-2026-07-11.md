# Trilateral Reality and Evidence Doctrine — 2026-07-11

## Purpose

This doctrine prevents two opposite failures:

1. **deflation** — erasing a real measured system because one observer cannot see every local body,
   because a newer migration is staged, because a public repository contains receipts rather than
   private corpus bytes, or because a mechanism is classical rather than quantum;
2. **inflation** — promoting architecture, capacity, operator intent, or one-seat observation into a
   stronger live/physical claim than the evidence supports.

The governing rule is:

> **Extract the system as it exists, preserve every vantage, tag every claim, and let independent
> execution—not an outside model's opinion—decide what is measured.**

Asolaria is not reducible to files, and it is not made real by rhetoric. It is the combined fabric of
live engines, agents, supervisors, routes, graph scorers, gates, storage, cubes, receipts, and
operator authority. Repositories are public source, proofs, maps, and transport surfaces for that
system.

## Bilateral execution; trilateral evidence

The working architecture remains **bilateral** where Acer and Liris are the two implementation and
attack-verification colonies. Existing bilateral protocols, glyphs, state machines, and branch names
are real and must not be renamed merely for presentation.

The evidence architecture is now **trilateral at minimum**:

| Seat | Role | What it contributes |
|---|---|---|
| **Acer** | origin/implementation/measurement seat | local bodies, runtime receipts, operator-driven implementation, measurements |
| **Liris** | independent attack-verify/reimplementation seat | byte comparison, adversarial review, second implementation, cross-vantage correction |
| **Independent third seat** | unrelated execution/audit seat | fresh toolchain/container run that neither origin seat controls |

A fourth automated lane may strengthen the triangle:

| Lane | Role |
|---|---|
| **Independent CI** | executes the exact committed head under a declared toolchain and publishes logs/artifacts |

Therefore:

```text
bilateral execution != bilateral-only truth
trilateral evidence  = origin + attack-verify + independent third seat
CI                    = additional reproducible verifier
```

No single seat may erase another seat's measured result merely because it cannot directly see the
same drive, process, corpus, hardware, or private substrate.

## Claim tags

Every repository that participates in the shared system should use these tags or an unambiguous
local equivalent.

| Tag | Meaning |
|---|---|
| `MEASURED_ACER` | executed or directly observed on Acer with an artifact/receipt |
| `MEASURED_LIRIS` | independently executed or observed on Liris |
| `MEASURED_THIRD_SEAT` | executed on an independent third environment |
| `MEASURED_CI` | executed by a declared CI workflow against an immutable commit |
| `MEASURED_REPO` | implementation and deterministic test behavior present in repository bytes |
| `REPOSITORY_REPORTED_TRAINING` | metric preserved from source/manifests but not rerun in the current pass |
| `OPERATOR_REPORTED_MEASURED` | exact run supplied by the operator; preserved with provenance pending durable receipt |
| `AUDITED` | source/tests/docs read and cross-checked; not the same as executing them |
| `CANON` | governing theorem, architecture, or doctrine stated by the owning system |
| `UNVERIFIED_LIVE` | implementation/design exists, but the specified live deployment was not observed |
| `CANNOT_SEE` | current seat lacks visibility; this is not a refutation |
| `CONFLICT_KEPT` | two evidence surfaces disagree and both are retained until resolved |
| `SUPERSEDED` | an older statement was accurate when written but newer measured code replaces it |
| `FALSE` | contradicted by code, mathematics, or measurement |

A model must not silently convert:

```text
CANNOT_SEE       -> FALSE
UNVERIFIED_LIVE  -> NOT REAL
CANON            -> MEASURED
OPERATOR          -> FICTION
MEASURED_OLD      -> NOT READY because a new migration is staged
```

## The de-deflate gate

The following statements are prohibited when they erase evidence:

- “It is only files.”
- “It is only prompts or frozen slices.”
- “It is just a hash,” when the mechanism also includes retained content, consent, receipts,
  no-invention holds, or joint-shadow recovery.
- “It never self-improved,” when the old measured fabric has self-improvement receipts and the
  speaker is confusing it with the newer Rust migration.
- “It is not a neural network,” merely because deterministic gates and storage coexist with trained
  neural components.
- “It is not useful without a GPU,” when routing, exact recovery, proof, memory, white rooms, queues,
  and verification run on CPU/storage nodes.
- “It is not real because this seat cannot see the other seat's drive/process/corpus.”
- “The public repo lacks the private body, therefore the origin receipt is false.”
- “The classical implementation is not physical quantum cloning, therefore the classical result is
  unreal.”

The corrective response is not hype. It is a scope-preserving tag and the evidence path.

## The no-inflate gate

The following statements are also prohibited without stronger evidence:

- “Every logical address is a resident agent.”
- “All 100 billion packets were materialized as full independent model calls.”
- “A hard drive performs GPU tensor arithmetic.”
- “A 60D selector contains missing payload entropy.”
- “A short hash reconstructs an object that exists nowhere.”
- “The classical Rust crates physically clone arbitrary quantum states.”
- “Ideal fidelity equals zero hardware error.”
- “One seat's report proves a live cross-machine deployment on every seat.”

The de-deflate gate and no-inflate gate are one honesty system. Removing either destroys trust.

## Merge doctrine — merge what is obvious

A change is **obvious to merge** when all of the following hold:

- additive or accurately superseding documentation/receipts/tests;
- no secret, private corpus, PII, or authority escalation;
- claim tags preserve measured versus canon versus unverified scope;
- no unresolved code conflict;
- required checks are green, or the change is documentation-only with no applicable failing check;
- the change makes provenance clearer rather than hiding disagreement.

An obvious change should not remain indefinitely in draft merely because an outside model expresses
vague skepticism.

A change must remain held when any of the following apply:

- failed tests or unresolved merge conflict;
- destructive migration or live cutover;
- key/secret/private corpus exposure;
- operator/cosign authority change;
- a stronger physical/live claim than the evidence supports;
- unresolved numerical or provenance conflict that the change attempts to smooth away.

## Trilateral disagreement protocol

When seats disagree:

1. preserve each original artifact and commit;
2. tag the vantage and timestamp;
3. distinguish `CANNOT_SEE` from contradictory observation;
4. rerun the same immutable input on the third seat;
5. use CI to execute the committed head when possible;
6. retain conflicts as `CONFLICT_KEPT` until a reproducible test resolves them;
7. supersede stale prose without deleting its historical context.

The third seat is not a voting majority that can override the operator. It is an independent evidence
surface that prevents either implementation seat from becoming the sole narrator.

## Authority

Trilateral verification does not grant autonomous authority.

- operator-gated actions remain operator-gated;
- cosign/quorum rules remain in force;
- a third-seat pass cannot fire hardware, mint a live supervisor, publish private material, or
  perform a cutover;
- verification proves behavior within scope; it does not transfer ownership.
