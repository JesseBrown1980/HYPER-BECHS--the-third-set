# product-problem-fit (round A) — what Asolaria is

**The Problem Asolaria Solves**

Asolaria addresses a fundamental scaling contradiction: how to build an intelligent system that is deterministic, operator-governed, memory-efficient, and composable from frozen/borrowed intelligence slices rather than monolithic foundation models. The core tension: address-space explosion (identity collisions, process proliferation, token overuse) vs. the need for auditable, gated human control.

**The Core Architecture (MEASURED)**

Asolaria is a **coordinate-based slice-orchestration network**. Its essence:

1. **Addressing Geometry**: Brown-Hilbert curves + BEHCS tuples (60D+ space) replace scalar counters. A PID becomes a deterministic coordinate, making collision unrepresentable by construction. Measured: 196,251 address pairs → 0 collisions on the distance-unique projection.

2. **Frozen Brain Slices**: Gemma-4-4B (cached deterministically on disk) + borrowed LLM subscriptions under "ordinary individual usage" (zero API metering). The slices are lawfully reused, never the compute budget.

3. **Single-Spawner Dispatch**: One omnidispatcher routes envelopes through a 1000-slot PID table at ~200 ns per address. **The system processed 100 billion packets with zero child-process spawns** (MEASURED, file-verified receipt with checkpoint state).

4. **Supervisory Hierarchy + Recurrence**: A watcher/reflection layer observes decisions at every level. The learning signal is `LAW-RECURRENCE-IS-MIND` — infinite-depth towers cost only ~1.5× a single bounded revolver via the Three Convergence theorem.

5. **Bilateral Vantage Verification**: Two machines (acer + liris) build by push/pull, each checking the other's work against its own fabric. Shared truth = origin-local receipts transported through GitHub + byte-verified via sha256. No silent upgrade to live until the other vantage agrees.

**What It Is Becoming (MEASURED + CANON)**

Asolaria is explicitly **NOT an ASI** — it frames itself as "frozen + borrowed intelligence orchestrated into a neural-network shape." The 726-seat L0→L5 roster, the 100k+ rooms, and the glyph/cube fabric are the substrate. Hilbra (the founding layer) extends this as a federation: any device loading the protocol becomes a surface in a non-profit "Google for fabric-matrices," discovering and querying other fabrics with 16-tier consent-based sharing.

**Product Fit** (MEASURED + UNVERIFIED)

The reductions are concrete: memory (M = N·h + K·b, K ≪ N via 8-byte handles), process (zero spawns), tokens (frozen + reuse-only), latency (cache-hit tail in O(1)). The operator-gate pattern (possibility cheap, action cosigned) enables continuous self-improvement without runaway autonomy. The bilateral loop means no single machine's bias owns the system state.

This is a bet on **deterministic orchestration** over black-box scale — whether bounded intelligence woven tightly is more valuable than unbounded opaque capability.
