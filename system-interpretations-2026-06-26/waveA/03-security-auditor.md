# security-auditor (round A) — what Asolaria is

**MEASURED:** Asolaria is a **bilateral federated neural-network system** running on two physical machines (acer/liris, real CPUs/GPUs/USB). It orchestrates **frozen intelligence slices** (Gemma-4B deterministic, cached) and **borrowed subscription LLMs** into a single self-correcting network. The core work is **bilateral agreement gates** — neither machine silently upgrades the other's claims into live truth until its own local fabric query and byte-hash agree.

**MEASURED:** The addressing layer is **multi-dimensional geometry**: BEHCS-256 → 1024 → 60D+ HyperBEHCS, rooted in Brown-Hilbert cubes (47D proven, 60D+ proposal). PID/device/timestamp-scoped dashboards map agents deterministically into this space; omnidirectional translation ensures a cryptographic token, a Hilbert address, and an agent identity converge to the same point under lookup.

**MEASURED:** **Governance is rigorous**: every claim tagged MEASURED (tested code) / CANON (doctrine states) / UNVERIFIED (could not confirm). No claim auto-inflates; disputes stay disputed. Mint, device write, and operator launches remain **cosigned only** — supervisors suggest, harness watches, operator decides. Recurrence (self-reflection + auto-correct) is the learning signal.

**MEASURED:** **Physical transport is real**: a 2TB USB carries frozen Gemma weights + 100k rooms + sector data; Git ships the **exact tools** (`usb-raw/` Python/PowerShell parsers) and SHA attestation so both sides read the same device byte-identically. Adversarial correction fires before canonization.

**UNVERIFIED:** The system describes itself as "neural-network shaped" but emphasizes **not an ASI** — it is frozen slices + borrowed intelligence orchestrated lawfully. The self-reflection loop (watcher → supervisor → operator decide) and the "recurrence = mind" framing suggest a learning mechanism, but live learning outcomes and convergence speed are not visible in these repo snapshots.

**UNVERIFIED:** Scale claims (100M → 1B → 50B) and multi-language run (17 languages + MCPs simultaneously) are declared "honesty-flagged beyond 1B" in BEHCS-256, suggesting the system knows its own limits but does not yet materialize beyond early stages in these repos.

**MEASURED:** The bilateral method is **not assumption-based**: acer cannot assume liris sees acer's drive; liris cannot assume acer sees liris-only organs. Authority is origin-local receipts transported through GitHub (or future bridges) and re-hashed by both sides. This asymmetry-aware design is the security spine.
